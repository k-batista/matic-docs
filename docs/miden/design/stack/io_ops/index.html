Input/Output Operations

In this section we describe the AIR constraints for Miden VM input / output operations. These operations move values between the stack and other components of the VM such as program code (i.e., decoder), memory, and advice provider.

PUSH​

The PUSH operation pushes the provided immediate value onto the stack (i.e., sets the value of s0s_0s0​ register). Currently, it is the only operation in Miden VM which carries an immediate value. The semantics of this operation are explained in the decoder section.

The effect of this operation on the rest of the stack is:

  • Right shift starting from position 000.

SDEPTH​

Assume aaa is the current depth of the stack stored in the stack bookkeeping register b0b_0b0​ (as described here). The SDEPTH pushes aaa onto the stack. The diagram below illustrates this graphically.

Stack transition for this operation must satisfy the following constraints:

s0′−b0=0 | degree=1s_0' - b_0 = 0 \text{ | degree} = 1s0′​−b0​=0 | degree=1

The effect of this operation on the rest of the stack is:

  • Right shift starting from position 000.

ADVPOP​

Assume aaa is an element at the top of the advice stack. The ADVPOP operation removes aaa from the advice stack and pushes it onto the operand stack. The diagram below illustrates this graphically.

The ADVPOP operation does not impose any constraints against the first element of the operand stack.

The effect of this operation on the rest of the operand stack is:

  • Right shift starting from position 000.

ADVPOPW​

Assume aaa, bbb, ccc, and ddd, are the elements at the top of the advice stack (with aaa being on top). The ADVPOPW operation removes these elements from the advice stack and puts them onto the operand stack by overwriting the top 444 stack elements. The diagram below illustrates this graphically.

The ADVPOPW operation does not impose any constraints against the top 444 elements of the operand stack.

The effect of this operation on the rest of the operand stack is:

  • No change starting from position 444.

Memory access operations​

Miden VM exposes several operations for reading from and writing to random access memory. Memory in Miden VM is managed by the Memory chiplet.

Communication between the stack and the memory chiplet is accomplished via the chiplet bus bchipb_{chip}bchip​. To make requests to the chiplet bus we need to divide its current value by the value representing memory access request. The structure of memory access request value is described here.

To enforce the correctness of memory access, we can use the following constraint:

bchip′⋅umem=bchip | degree=2b_{chip}' \cdot u_{mem} = b_{chip} \text{ | degree} = 2bchip′​⋅umem​=bchip​ | degree=2

In the above, umemu_{mem}umem​ is the value of memory access request. Thus, to describe AIR constraint for memory operations, it is sufficient to describe how umemu_{mem}umem​ is computed. We do this in the following sections.

MLOADW​

Assume that the word with elements v0,v1,v2,v3v_0, v_1, v_2, v_3v0​,v1​,v2​,v3​ is located in memory at address aaa. The MLOADW operation pops an element off the stack, interprets it as a memory address, and replaces the remaining 4 elements at the top of the stack with values located at the specified address. The diagram below illustrates this graphically.

To simplify description of the memory access request value, we first define a variable for the value that represents the state of memory after the operation:

v=∑i=03αi+5⋅s3−i′v = \sum_{i=0}^3\alpha_{i+5} \cdot s_{3-i}'v=i=0∑3​αi+5​⋅s3−i′​

Using the above variable, we define the value representing the memory access request as follows:

umem=α0+α1⋅opmem_read+α3⋅s0+α4⋅clk+vu_{mem} = \alpha_0 + \alpha_1 \cdot op_{mem\_read} + \alpha_3 \cdot s_0 + \alpha_4 \cdot clk + vumem​=α0​+α1​⋅opmem_read​+α3​⋅s0​+α4​⋅clk+v

  • s0s_0s0​ is the memory address from which the values are to be loaded onto the stack.
  • clkclkclk is the current clock cycle of the VM.

  • Left shift starting from position 555.

The MLOAD operation pops an element off the stack, interprets it as a memory address, and pushes the first element of the word located at the specified address to the stack. MSTOREW​

The MSTOREW operation pops an element off the stack, interprets it as a memory address, and writes the remaining 444 elements at the top of the stack into memory at the specified address. The stored elements are not removed from the stack. The diagram below illustrates this graphically.

After the operation the contents of memory at address aaa would be set to v0,v1,v2,v3v_0, v_1, v_2, v_3v0​,v1​,v2​,v3​.

To simplify description of the memory access request value, we first define a variable for the value that represents the state of memory after the operation:

umem=α0+α1⋅opmem_write+α3⋅s0+α4⋅clk+vu_{mem} = \alpha_0 + \alpha_1 \cdot op_{mem\_write} + \alpha_3 \cdot s_0 + \alpha_4 \cdot clk + vumem​=α0​+α1​⋅opmem_write​+α3​⋅s0​+α4​⋅clk+v

In the above:

  • s0s_0s0​ is the memory address into which the values from the stack are to be saved.
  • clkclkclk is the current clock cycle of the VM.

The effect of this operation on the rest of the stack is:

  • Left shift starting from position 111.

The MSTORE operation pops an element off the stack, interprets it as a memory address, and writes the remaining element at the top of the stack into the first element of the word located at the specified memory address. The remaining 333 elements of the word are not affected. The diagram below illustrates this graphically.
