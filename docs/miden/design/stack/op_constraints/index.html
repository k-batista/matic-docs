Stack Operation Constraints

In addition to the constraints described in the previous section, we need to impose constraints to check that each VM operation is executed correctly.

For this purpose the VM exposes a set of operation-specific flags. These flags are set to 111 when a given operation is executed, and to 000 otherwise. The naming convention for these flags is fopnamef_{opname}fopname​. For example, fdupf_{dup}fdup​ would be set to 111 when DUP operation is executed, and to 000 otherwise. Operation flags are discussed in detail in the section below.

To describe how operation-specific constraints work, let's use an example with DUP operation. This operation pushes a copy of the top stack item onto the stack. The constraints we need to impose for this operation are as follows:

fdup⋅(s0′−s0)=0fdup⋅(si+1′−si)=0  for i∈{0,..,14}f_{dup} \cdot (s'_0 - s_0) = 0 \\ f_{dup} \cdot (s'_{i+1} - s_i) = 0 \ \text{ for } i \in \{0, .., 14\}fdup​⋅(s0′​−s0​)=0fdup​⋅(si+1′​−si​)=0  for i∈{0,..,14}

The first constraint enforces that the top stack item in the next row is the same as the top stack item in the current row. The second constraint enforces that all stack items (starting from item 000) are shifted to the right by 111. We also need to impose all the constraints discussed in the previous section, be we omit them here.

Let's write similar constraints for DUP1 operation, which pushes a copy of the second stack item onto the stack:

fdup1⋅(s0′−s1)=0fdup1⋅(si+1′−si)=0  for i∈{0,..,14}f_{dup1} \cdot (s'_0 - s_1) = 0 \\ f_{dup1} \cdot (s'_{i+1} - s_i) = 0 \ \text{ for } i \in \{0, .., 14\}fdup1​⋅(s0′​−s1​)=0fdup1​⋅(si+1′​−si​)=0  for i∈{0,..,14}

It is easy to notice that while the first constraint changed, the second constraint remained the same - i.e., we are still just shifting the stack to the right.

In fact, for most operations it makes sense to make a distinction between constraints unique to the operation vs. more general constraints which enforce correct behavior for the stack items not affected by the operation. In the subsequent sections we describe in detail only the former constraints, and provide high-level descriptions of the more general constraints. Specifically, we indicate how the operation affects the rest of the stack (e.g., shifts right starting from position 000).

Operation flags​

As mentioned above, operation flags are used as selectors to enforce operation-specific constraints. That is, they turn on relevant constraints for a given operation. In total, the VM provides 888888 unique operations, and thus, there are 888888 operation flags (not all of them currently used).

Operation flags are mutually exclusive. That is, if one flag is set to 111, all other flags are set to 000. Also, one of the flags is always guaranteed to be set to 111.

To compute values of operation flags we use op bits registers located in the decoder. These registers contain binary representations of operation codes (opcodes). Each opcode consists of 777 bits, and thus, there are 777 op bits registers. We denote these registers as b0,...,b6b_0, ..., b_6b0​,...,b6​. The values are computed by multiplying the op bit registers in various combinations.

For example, the value of the flag for NOOP, which is encoded as 0000000, is computed as follows:

fnoop=(1−b0)⋅(1−b1)⋅(1−b2)⋅(1−b3)⋅(1−b4)⋅(1−b5)⋅(1−b6)f_{noop} = (1 - b_0) \cdot (1 - b_1) \cdot (1 - b_2) \cdot (1 - b_3) \cdot (1 - b_4) \cdot (1 - b_5) \cdot (1 - b_6)fnoop​=(1−b0​)⋅(1−b1​)⋅(1−b2​)⋅(1−b3​)⋅(1−b4​)⋅(1−b5​)⋅(1−b6​)

While the value of the DROP operation, which is encoded as 0101001 is computed as follows:

fdrop=(1−b0)⋅b1⋅(1−b2)⋅b3⋅(1−b4)⋅(1−b5)⋅b6f_{drop} = (1 - b_0) \cdot b_1 \cdot (1 - b_2) \cdot b_3 \cdot (1 - b_4) \cdot (1 - b_5) \cdot b_6fdrop​=(1−b0​)⋅b1​⋅(1−b2​)⋅b3​⋅(1−b4​)⋅(1−b5​)⋅b6​

As can be seen from above, the degree for both of these flags is 777. Since degree of constraints in Miden VM can go up to 999, this means that operation-specific constraints cannot exceed degree 222. However, there are some operations which require constraints of higher degree (e.g., 333 or even 555). To support such constraints, we adopt the following scheme.

We organize the operations into 333 groups as shown below and also introduce an extra register for degree reduction:

b6b_6b6​b5b_5b5​b4b_4b4​b3b_3b3​b2b_2b2​b1b_1b1​b0b_0b0​extra# of opsdegree
0xxxxxx0647
10xxxx-0166
11xxx--184

In the above:

  • Operation flags for operations in the first group (with prefix 0), are computed using all 777 op bits, and thus their degree is 777.
  • Operation flags for operations in the second group (with prefix 10), are computed using only the first 666 op bits, and thus, their degree is 666.
  • Operation flags for operations in the third group (with prefix 11), are computed using only the first 555 op bits, and we also use the extra register (which is set to b6⋅b5b_6 \cdot b_5b6​⋅b5​) to reduce the degree by 111. Thus, the degree of op flags in this group is 444.

How operations are distributed between these 333 groups is described in the sections below.

u32 operations​

This group contains 888 u32 operations. These operations are grouped together because all of them require range checks. The constraints for range checks are of degree 555, however, since all these operations require them, we can define a flag with common prefix 100 to serve as a selector for the range check constraints. The value of this flag is computed as follows:

fu32rc=b6⋅(1−b5)⋅(1−b4)f_{u32rc} = b_6 \cdot (1 - b_5) \cdot (1 - b_4)fu32rc​=b6​⋅(1−b5​)⋅(1−b4​)

The degree of this flag is 333, which is acceptable for a selector for degree 555 constraints.

OperationOpcode valueBinary encodingOperation groupFlag degree
U32ADD646464100_0000u32 ops666

As mentioned previously, the last bit of the opcode is not used in computation of the flag for these operations. We force this bit to always be set to 000 with the following constraint:

b6⋅(1−b5)⋅b0=0 | degree=3b_6 \cdot (1 - b_5) \cdot b_0 = 0 \text{ | degree} = 3b6​⋅(1−b5​)⋅b0​=0 | degree=3

Putting these operations into a group with flag degree 666 is important for two other reasons:

  • Constraints for U32SPLIT operation have degree 333, and thus, degree of op flag for this operation cannot exceed 666.
  • Operations U32ADD3 and U32MADD shift the stack to the left. Thus, having these two operations in this group and putting them under the common prefix 10011, allows us to create a common flag for these operations of degree 555 (recall that left-shift flag cannot exceed degree 555).

Composite flags​

Using the operation flags defined above, we can compute several composite flags which are used by various constraints in the VM.

Shift right flag​

The right-shift flag indicates that an operation shifts the stack to the right. This flag is computed as follows:

fshr=(1−b6)⋅b5⋅b4+fu32split+fpush | degree=6f_{shr} = (1 - b_6) \cdot b_5 \cdot b_4 + f_{u32split} + f_{push} \text{ | degree} = 6fshr​=(1−b6​)⋅b5​⋅b4​+fu32split​+fpush​ | degree=6

In the above, (1−b6)⋅b5⋅b4(1 - b_6) \cdot b_5 \cdot b_4(1−b6​)⋅b5​⋅b4​ evaluates to 111 for all right stack shift operations described previously. This works because all these operations have a common prefix 011. We also need to add in flags for other operations which shift the stack to the right but are not a part of the above group (e.g., PUSH operation).

Shift left flag​

The left-shift flag indicates that a given operation shifts the stack to the left. To simplify the description of this flag, we will first compute the following intermediate variables:

A flag which is set to 111 when fu32add3=1f_{u32add3} = 1fu32add3​=1 or fu32madd=1f_{u32madd} = 1fu32madd​=1:

fadd3_madd=b6⋅(1−b5)⋅(1−b4)⋅b3⋅b2 | degree=5f_{add3\_madd} = b_6 \cdot (1 - b_5) \cdot (1 - b_4) \cdot b_3 \cdot b_2 \text{ | degree} = 5fadd3_madd​=b6​⋅(1−b5​)⋅(1−b4​)⋅b3​⋅b2​ | degree=5

A flag which is set to 111 when fsplit=1f_{split} = 1fsplit​=1 or floop=1f_{loop} = 1floop​=1:

fsplit_loop=b6⋅(1−b5)⋅b4⋅b3⋅b2 | degree=5f_{split\_loop} = b_6 \cdot (1 - b_5) \cdot b_4 \cdot b_3 \cdot b_2 \text{ | degree} = 5fsplit_loop​=b6​⋅(1−b5​)⋅b4​⋅b3​⋅b2​ | degree=5

Using the above variables, we compute left-shift flag as follows:

fshl=(1−b6)⋅b5⋅(1−b4)+fadd3_madd+fsplit_loop+frepeat+fend⋅h5 | degree=5f_{shl} = (1 - b_6) \cdot b_5 \cdot (1 - b_4) + f_{add3\_madd} + f_{split\_loop} + f_{repeat} + f_{end} \cdot h_5 \text{ | degree} = 5fshl​=(1−b6​)⋅b5​⋅(1−b4​)+fadd3_madd​+fsplit_loop​+frepeat​+fend​⋅h5​ | degree=5

In the above:

  • (1−b6)⋅b5⋅(1−b4)(1 - b_6) \cdot b_5 \cdot (1 - b_4)(1−b6​)⋅b5​⋅(1−b4​) evaluates to 111 for all left stack shift operations described previously. This works because all these operations have a common prefix 010.
  • h5h_5h5​ is the helper register in the decoder which is set to 111 when we are exiting a LOOP block, and to 000 otherwise.

Thus, similarly to the right-shift flag, we compute the value of the left-shift flag based on the prefix of the operation group which contains most left shift operations, and add in flag values for other operations which shift the stack to the left but are not a part of this group.

Control flow flag​

The control flow flag fctrlf_{ctrl}fctrl​ is set to 111 when a control flow operation is being executed by the VM, and to 000 otherwise. Naively, this flag can be computed as follows:

fctrl=fjoin+fsplit+floop+frepeat+fspan+frespan+fcall+fsyscall+fend+fhalt | degree=6f_{ctrl} = f_{join} + f_{split} + f_{loop} + f_{repeat} + f_{span} + f_{respan} + f_{call} + f_{syscall} + f_{end} + f_{halt} \text{ | degree} = 6fctrl​=fjoin​+fsplit​+floop​+frepeat​+fspan​+frespan​+fcall​+fsyscall​+fend​+fhalt​ | degree=6

However, this can be computed more efficiently via the common operation prefixes for the two groups of control flow operations as follows.

fspan,join,split,loop=b6⋅(1−b5)⋅b4⋅b3 | degree=4f_{span,join,split,loop} = b_6 \cdot (1 - b_5) \cdot b_4 \cdot b_3 \text{ | degree} = 4fspan,join,split,loop​=b6​⋅(1−b5​)⋅b4​⋅b3​ | degree=4
fend,repeat,respan,halt=b6⋅b5⋅b4 | degree=3f_{end,repeat,respan,halt} = b_6 \cdot b_5 \cdot b_4 \text{ | degree} = 3fend,repeat,respan,halt​=b6​⋅b5​⋅b4​ | degree=3
fctrl=fspan,join,split,loop+fend,repeat,respan,halt+fcall+fsyscall | degree=4f_{ctrl} = f_{span,join,split,loop} + f_{end,repeat,respan,halt} + f_{call} + f_{syscall} \text{ | degree} = 4fctrl​=fspan,join,split,loop​+fend,repeat,respan,halt​+fcall​+fsyscall​ | degree=4

Note that the degree of fend,repeat,respan,haltf_{end,repeat,respan,halt}fend,repeat,respan,halt​ can be reduced to degree 2 using the extra column, but this will not affect the degree of the fctrlf_{ctrl}fctrl​ constraint.
