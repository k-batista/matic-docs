Wallet Widget

Wallet widget is a UI tool which can be embedded in any web application for executing bridge transactions - Deposit & Withdraw.

Every widget is identified by an unique name which you can get from Widget dashboard .

Widget dashboardâ€‹

Widget can be created from the widget dashboard page in the wallet application. It allows the user to create a new widget with some customisable options.

Once the widget is created, You can copy code snippet and add it in your application or use the widget name and configure by yourself.

Here is link to widget dashboard -

  • mainnet - https://wallet.polygon.technology/widget-dashboard
  • testnet - https://wallet-dev.polygon.technology/widget-dashboard

Installâ€‹

Widget is exported as javascript library and available as npm package. 

npm i @maticnetwork/wallet-widget

Examplesâ€‹

We have created examples for different framework and tools to help you with the development. All examples are present at - https://github.com/maticnetwork/wallet-widget-example

How to useâ€‹

With targetâ€‹

Consider you have a button in your app and you want to show widget when clicked on that button - 

<button id="btnMaticWidget"></btn>
import { Widget } from "@maticnetwork/wallet-widget";

var widget = new Widget({
    appName: "<widget name>", //widget name from dashboard
    target: '#btnMaticWidget', // element selector for showing widget on click
    network: 'mainnet' // network to be used - testnet or mainnet
});

Create widget whenever you are ready. It is best to call create function after document is loaded.

await widget.create();

widget is created, now click on your button and widget will be shown.

Without targetâ€‹

import { Widget } from "@maticnetwork/wallet-widget";

var widget = new Widget({
    appName: "<widget name>", //widget name from dashboard
    network: 'mainnet' // network to be used - testnet or mainnet
});

await widget.create();

widget is now created, but in order to show the widget - you will have to call show API.

widget.show();

Similarly you can hide the widget, by calling hide API.

widget.hide();

Important Note ðŸ‘‰â€‹

  1. Based on network "testnet" or "mainnet", you need to create your app on respective dashboard. We recommend to create app with same name on both testnet & mainnet, so that you don't have any issue when you are changing network.

  2. Wallet widget is UI Library and on different website it will look different & might have some issues like - colors, responsiveness etc. So please spend some time on testing and customizing. In case of any help needed - please reach out to support team.

  3. Wallet widget is full screen in mobile devices but you can customize it by style configuration.

Configurationâ€‹

Configuration can be supplied in Widget constructor.

Available configuration areâ€‹

  • targetÂ : string - CSS selector for showing widget on click of element. For example, "#btnMaticWidget" will be the target in the code below.
<button id="btnMaticWidget">Matic widget</button>
  • networkÂ : string - network to be used. Two options are available - 'testnet' or 'mainnet'.
  • widthÂ : number - Width of the widget
  • heightÂ : number - Height of the widget
  • autoShowTimeÂ : number - Auto show widget after specified time in millisecond
  • appNameÂ : string - name of your app, this can be retrieved on widget dashboard.
  • positionÂ : string - Sets the position of the widget. The available options are -
    • center
    • bottom-right
    • bottom-left
  • amountÂ : string - Prefill the amount in text box
  • page : string - select the page. Available options are - withdraw, deposit.
  • overlay : boolean - show overlay when widget is opened. By default it is false.
  • style : object - apply some css styles to the widget. 
var widget = new MaticWidget({
    appName: "<your app id>", //appName from dashboard
    target: '#btnMaticWidget', // element selector for showing widget on click
    network: 'testnet' // network to be used - testnet or mainnet,
    style:{
      color:'red'
    }
});

Eventsâ€‹

Widget emits some events which can be used to know what is happening inside the application.

Subscribe to eventsâ€‹

widget.on('load',()=>{
  console.log('widget is loaded');
})

Unsubscribe to eventsâ€‹

widget.off('load',<callback>)

Callback should be same as what was used to subscribe the event. So its better to store the callback in a variable.`

List of events:â€‹

  • loadÂ - Widget is loaded
  • closeÂ - Widget is closed
  • approveInitÂ - Approval transaction is initialized
  • approveCompleteÂ - Approval transaction is completed
  • approveErrorÂ - Approval transaction failed due to some error, or the user denied the transaction on Metamask
  • depositInitÂ - Deposit transaction is initialized
  • depositCompleteÂ - Deposit transaction is completed
  • depositErrorÂ - Deposit transaction failed due to some error, or the user denied the deposit complete transaction on Metamask
  • burnInitÂ - Withdrawal burn transaction is initialized
  • burnCompleteÂ - Withdrawal burn transaction is completed
  • confirmWithdrawInitÂ - Withdrawal is checkpointed and confirm transaction is initialized
  • confirmWithdrawCompleteÂ - Withdrawal confirm transaction in completed
  • confirmWithdrawErrorÂ - Withdrawal confirm transaction failed due to some error, or the user denied the withdrawal confirm transaction on Metamask
  • exitInitÂ - Withdrawal exit transaction is initialized
  • exitCompleteÂ - Withdrawal exit transaction is completed
  • exitErrorÂ - Withdrawal exit transaction failed due to some error, or the user denied the withdrawal exit transaction on Metamask

APISâ€‹

  • show - show the widget
widget.show()
  • hide - hide the widget
widget.hide()
  • on - subscribe to events
widget.on('<event name>', callback)
  • off - unsubscribe to events
widget.off('<event name>', callback)
